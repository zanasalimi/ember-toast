# Media capture guide

This folder holds the showcase media (GIFs/video) referenced from the root
`README.md`. The clips can't be generated by tooling — they're screen recordings
of the live playground — so this file specifies exactly **what to capture**,
**how**, and **where each file is referenced**, so the recordings can be dropped
in without touching the markup.

Record against the docs site (`pnpm docs:dev`, then `http://localhost:3000`). Use
the in-page **live playground** for every clip. Keep the viewport tidy (light
theme, the editorial page behind the toasts) and the cursor visible where a
gesture is shown.

## Files to produce

Drop the finished files at these exact paths (the README already links them as
placeholders):

| File | Format | ~Length | What it shows |
|---|---|---|---|
| `hero.gif` | GIF (or `hero.mp4`) | 4–6s loop | The signature loop: fire Success, Error, and Info in quick succession so a stack of 3 builds, then let one auto-dismiss. Seamless loop. This is the README hero. |
| `flip-reflow.gif` | GIF | 3–4s | The FLIP reflow. Build a stack of 3+, then dismiss the **top/middle** one and let the survivors **slide** up into the freed space (not jump). Slow-ish so the slide is legible. |
| `promise-morph.gif` | GIF | 3–4s | `toast.promise`. Click **Promise ✓**: one toast goes loading → success **in place** (same toast, no flit). Optionally a second take with **Promise ✕** for the error branch. |
| `swipe-out.gif` | GIF | 2–3s | Swipe-to-dismiss. Press-drag a toast past the threshold and release so it flings out; show one under-threshold drag that snaps back. Mouse drag is fine; a touch capture reads better. |
| `walkthrough.mp4` | MP4 (H.264) | ~75s | Narrated/sped tour: install line → fire each variant → change `position`/`theme`/`visibleToasts`/`richColors` live → promise → swipe → reduced-motion note. The "30-second tour" link in the README points here. |

## How to capture

- **GIFs:** record with any screen recorder, then convert. Keep them small —
  target < 4 MB each (the hero especially). A reliable recipe:
  - record to `.mp4` (e.g. ScreenToGif, OBS, or QuickTime)
  - `ffmpeg -i in.mp4 -vf "fps=24,scale=900:-1:flags=lanczos" -c:v gif out.gif`
  - or keep `.mp4` and update the README `<img>`/`<video>` accordingly.
- **Dimensions:** ~900px wide is plenty for a README; the toasts are 356px each.
- **Reduced motion:** for the walkthrough's reduced-motion beat, toggle the OS
  "reduce motion" setting and show a dismiss happening instantly.
- **Accessibility:** keep meaningful alt text on each embed (already set in the
  README placeholders).
